1 Pendahuluan
1.1 Latar Belakang

BPS Kabupaten Minahasa Selatan sebelumnya menggunakan platform AppSheet berbasis Google Sheets untuk memantau aktivitas harian pegawai. Pendekatan ini memiliki sejumlah keterbatasan operasional:

  • ✗ Ketergantungan penuh pada koneksi internet untuk setiap aksi
  • ✗ Tidak ada pemisahan hak akses yang granular antar pegawai
  • ✗ Performa lambat saat data bertambah banyak
  • ✗ Tampilan tidak optimal di perangkat mobile
  • ✗ Tidak ada mekanisme caching gambar — setiap buka halaman mengunduh ulang semua foto

Berdasarkan kondisi tersebut, dikembangkan aplikasi PantauPegawai sebagai pengganti yang dibangun secara native menggunakan Flutter, dengan backend Supabase dan penyimpanan file di Google Drive.

1.2 Tujuan
No Tujuan Indikator Keberhasilan
1 Menyediakan platform pencatatan aktivitas harian pegawai yang mudah digunakan Pegawai dapat input dokumentasi dalam < 1 menit
2 Memungkinkan admin memantau seluruh aktivitas pegawai secara real-time Dashboard menampilkan data terkini tanpa refresh manual
3 Mengurangi penggunaan data internet dengan image caching Gambar yang sudah diunduh tidak diunduh ulang
4 Menyimpan dokumentasi foto secara terorganisir di Google Drive Foto tersimpan dalam struktur folder pegawai/tanggal
5 Menerapkan keamanan akses data berbasis role Pegawai tidak dapat mengakses data pegawai lain
1.3 Ruang Lingkup

✓ Dalam Lingkup

  • • Aplikasi mobile Android untuk pegawai
  • • Aplikasi web untuk admin
  • • Pencatatan dokumentasi harian dengan foto
  • • Manajemen proyek/kegiatan
  • • Dashboard statistik admin
  • • Upload dan penyimpanan foto ke Google Drive
  • • Rekap upload per pegawai

✗ Di Luar Lingkup

  • • Aplikasi iOS (belum dalam rencana)
  • • Sistem penggajian
  • • Absensi berbasis GPS/biometrik
  • • Notifikasi push
  • • Integrasi SIMPEG BPS Pusat
2 Analisis Kebutuhan
2.1 Kebutuhan Fungsional
No Kode Kebutuhan Aktor Prioritas
1 KF-01 Login dengan email dan password Admin, Pegawai Tinggi
2 KF-02 Redirect otomatis berdasarkan role setelah login Sistem Tinggi
3 KF-03 Input dokumentasi harian: foto, proyek, tanggal, catatan Pegawai Tinggi
4 KF-04 Upload foto ke Google Drive secara otomatis Sistem Tinggi
5 KF-05 Tampil daftar dokumentasi harian dengan filter tanggal dan pegawai Admin, Pegawai Tinggi
6 KF-06 Dashboard statistik: jumlah proyek, total dokumentasi, total pegawai Admin Tinggi
7 KF-07 Manajemen proyek/kegiatan (tambah, edit, hapus) Admin Tinggi
8 KF-08 Dropdown proyek dinamis pada form dokumentasi Pegawai Tinggi
9 KF-09 Tombol Copy Link untuk menyalin URL Drive gambar Admin, Pegawai Sedang
10 KF-10 Manajemen pegawai: tambah, edit, reset password, hapus Admin Tinggi
11 KF-11 Rekap upload dokumentasi per pegawai Admin Sedang
12 KF-12 Import data dari Google Sheets Admin Rendah
13 KF-13 Hapus dokumentasi milik sendiri Pegawai Sedang
14 KF-14 Lihat riwayat dokumentasi pribadi Pegawai Tinggi
2.2 Kebutuhan Non-Fungsional
No Kategori Kebutuhan Target
1 Performa Waktu muat halaman utama < 3 detik pada jaringan 4G
2 Performa Image caching — gambar yang sudah diunduh tidak diunduh ulang 0 request jaringan untuk gambar yang sudah di-cache
3 Keamanan Isolasi data antar pegawai menggunakan RLS Pegawai tidak dapat membaca/mengubah data pegawai lain
4 Keamanan Credentials Google Service Account tidak tersimpan di client Hanya ada di Edge Function secrets
5 Ketersediaan Uptime backend Supabase 99.9% (SLA Supabase)
6 Kompatibilitas Platform Android minimum Android 5.0 (API 21)
7 Usability Input dokumentasi dapat diselesaikan tanpa pelatihan khusus Onboarding < 5 menit
8 Skalabilitas Jumlah pegawai yang dapat ditangani Hingga 100 pegawai aktif
2.3 Identifikasi Pengguna

👔 Admin

  • Melihat dashboard statistik seluruh pegawai
  • Mengelola daftar proyek/kegiatan
  • Mengelola akun pegawai (CRUD + reset password)
  • Melihat semua dokumentasi harian
  • Filter dokumentasi per pegawai dan tanggal
  • Melihat rekap upload
  • Import data dari Google Sheets
  • Mengakses via Flutter Web (browser)

👤 Pegawai

  • Login dengan akun yang dibuat admin
  • Input dokumentasi harian (foto + proyek + catatan)
  • Memilih proyek dari dropdown yang disediakan admin
  • Melihat riwayat dokumentasi pribadi
  • Menghapus dokumentasi milik sendiri
  • Menyalin link Drive foto dengan tombol Copy Link
  • Mengakses via aplikasi Android
