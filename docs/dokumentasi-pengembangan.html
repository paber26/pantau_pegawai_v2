Dokumentasi Pengembangan Aplikasi Monitoring Aktivitas Pegawai

📅 Tahun 2026 🏛️ BPS Kabupaten Minahasa Selatan 📱 Flutter + Supabase
Tahapan Pengembangan
1
🔍
Analisis Kebutuhan
Identifikasi proses bisnis & kebutuhan pengguna

Dilakukan identifikasi kebutuhan fungsional dan non-fungsional melalui observasi proses monitoring aktivitas pegawai yang berjalan secara manual, serta wawancara dengan pengguna untuk memahami alur kerja yang diharapkan.

  • Identifikasi peran pengguna: Admin dan Pegawai
  • Pemetaan alur pencatatan aktivitas harian pegawai
  • Penentuan kebutuhan penyimpanan dokumentasi foto kegiatan
  • Analisis kebutuhan laporan dan rekap data
  • Penetapan daftar proyek/kegiatan yang dipantau
2
🗂️
Perancangan Sistem
Desain database, workflow, dan antarmuka

Perancangan arsitektur sistem mencakup skema database di Supabase (PostgreSQL), alur autentikasi berbasis role, desain antarmuka untuk Admin (web) dan Pegawai (mobile), serta integrasi penyimpanan file ke Google Drive.

  • Desain skema tabel: users , kegiatan , dokumentasi , penugasan
  • Penerapan Row Level Security (RLS) untuk isolasi data antar pegawai
  • Perancangan alur upload foto via Supabase Edge Function ke Google Drive
  • Desain wireframe halaman Admin (Dashboard, Proyek, Dokumentasi)
  • Desain wireframe halaman Pegawai (Form dokumentasi, Riwayat)
3
⚙️
Pengembangan
Implementasi fitur-fitur utama aplikasi

Pengembangan dilakukan menggunakan Flutter dengan state management Riverpod dan navigasi go_router. Backend menggunakan Supabase sebagai database dan autentikasi, serta Supabase Edge Function (Deno/TypeScript) sebagai proxy upload ke Google Drive.

  • Autentikasi login berbasis role (Admin / Pegawai)
  • Pencatatan dokumentasi harian: foto, proyek, tanggal, catatan
  • Upload foto ke Google Drive via Edge Function dengan image caching lokal
  • Dashboard Admin: statistik jumlah proyek, total dokumentasi, pegawai
  • Manajemen proyek/kegiatan oleh Admin dengan bulk import
  • Dropdown proyek dinamis pada form dokumentasi pegawai
  • Tombol Copy Link untuk menyalin URL Drive file gambar
  • Rekap upload dan filter dokumentasi per pegawai/tanggal
  • Integrasi logo BPS sebagai identitas visual dan launcher icon
4
🧪
Pengujian
Verifikasi fungsi dan kesesuaian kebutuhan

Pengujian dilakukan untuk memastikan seluruh fitur berjalan sesuai kebutuhan, mencakup pengujian fungsional alur utama, pengujian antarmuka pada perangkat Android dan browser web, serta validasi keamanan akses data antar pengguna.

  • Pengujian alur login dan pemisahan hak akses Admin vs Pegawai
  • Pengujian upload foto dan penyimpanan ke Google Drive
  • Verifikasi image caching — gambar tidak diunduh ulang jika sudah tersimpan
  • Pengujian validasi form dokumentasi (proyek wajib dipilih)
  • Pengujian fungsi Copy Link pada kartu dokumentasi
  • Pengujian tampilan responsif di Android dan Flutter Web
5
🚀
Implementasi & Evaluasi
Deployment dan penyempurnaan berdasarkan masukan

Aplikasi di-deploy pada lingkungan operasional: APK Android didistribusikan ke pegawai, antarmuka Admin diakses melalui browser web. Evaluasi dilakukan berdasarkan masukan pengguna untuk penyempurnaan berkelanjutan.

  • Build APK release Flutter untuk distribusi ke perangkat Android pegawai
  • Deployment Flutter Web untuk akses Admin via browser
  • Import 65 data proyek BPS Minahasa Selatan ke database
  • Penyempurnaan UI: hapus field tidak relevan, sederhanakan tampilan kartu proyek
  • Evaluasi dan iterasi berdasarkan masukan pengguna lapangan
